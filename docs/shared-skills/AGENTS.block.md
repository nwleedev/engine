## Shared Skills

- Treat this block as a routing shim: when a listed workflow applies, invoke the matching shared-skills skill and follow its `SKILL.md`.
- Use `requirements-packet`, `spec-contract`, `plan-contract`, and `spec-plan-coverage` when requirements, research, specs, plans, tests, implementation evidence, or completion claims need traceability.
- Use `research-plan`, `source-ledger`, and `claim-evidence-map` for source-backed research, evidence ledgers, claim confidence, and counterevidence.
- Route behavior-changing tests through `scenario-test-designer`, `test-plan-contract`, and `tdd-test-writing`; send infeasible tests to `verification-gate`.
- Use `implementation-discipline`, `debugging-discipline`, `comment-writing`, `implementation-evidence`, and `verification-gate` for scoped execution, investigation, comments, evidence, and completion claims.
- Keep detailed workflows in the installed shared-skills `SKILL.md` and `references/*`; use project-local rules when shared-skills is absent, and use the install/status diagnostic.

## Shared Skills

- Treat this block as a routing shim: when a listed workflow applies, invoke the matching shared-skills skill and follow its `SKILL.md`.
- Use `requirements-packet`, `spec-contract`, `plan-contract`, and `spec-plan-coverage` when requirements, research, specs, plans, tests, implementation evidence, or completion claims need traceability.
- Route behavior-changing tests through `scenario-test-designer`, `test-plan-contract`, and `tdd-test-writing`; record infeasible automated tests and residual risk in `verification-gate`.
- Use `implementation-evidence` and `verification-gate` before claiming work is complete, correct, ready, or decision-ready.
- Keep detailed workflows in the installed shared-skills `SKILL.md` and `references/*`; do not duplicate them in AGENTS.md.
- Use project-local rules when shared-skills is not installed, and use the install/status diagnostic to check whether this compact block is present and not over-expanded.

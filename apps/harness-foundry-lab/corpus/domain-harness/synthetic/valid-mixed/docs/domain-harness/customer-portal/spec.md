# Customer Portal Harness

## Scope

This harness supports application work and product planning.

## Development guardrails

- Implementation scope must name allowed modules.
- Test strategy must include regression checks.
- Security review must cover auth and data exposure.
- Dependency changes require explicit approval.

## Non-development guardrails

- Source quality must separate evidence from assumptions.
- Privacy review must remove customer details.
- Tone guidance must match product communication standards.
- Approval flow must be documented before external use.

## Mixed split guardrails

- Development tasks and non-development tasks must use separate acceptance criteria.
- Research findings must not automatically authorize code changes.
